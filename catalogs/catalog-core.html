Core Data Catalog

The NCCS Core Data Series is a panel of nonprofit organizations derived from annual IRS Form 990 filings. This catalog organizes the published files by form type — pick the variant whose filing scope matches your analysis.

For background on what the Core Series contains and its scope limitations, see the Core overview page.

NoteFiles are organized by tax year, not filing year

Every year label in this catalog refers to the tax year covered by the return — the fiscal period the filing describes — not the calendar year the IRS received it. A nonprofit’s 2022 tax-year return is in the 2022 directory regardless of when it was actually filed (typically 6–18 months later). Recent tax years are still in-progress as the IRS continues to release returns over time.

Which dataset should I use?

If you need… Use Why
Full Form 990 filers only, max variables 990 Larger nonprofits filing the long form (251 columns in 2024). Best for detailed financial analysis.
Short-form filers only 990EZ Smaller nonprofits filing the short form (76 columns in 2024).
Every 990 + 990EZ filer in one file 990 Combined The broadest population per year. Stacks 990 and 990EZ filers using the 53-column shared variable subset (plus pipeline-derived universals: tax_year, tax_month, is_501c3, extract_year, is_amendment).
Private foundations 990PF Filed by all private foundations regardless of size. Separate variable set from public charities (187 columns in 2024).

Each file pools 501(c)(3) charities and all other 501(c) types into a single product. Use the is_501c3 column (boolean, TRUE for 501(c)(3) public charities) to filter when needed; the raw subsection_cd (IRC EO subsection code, 1–93) is also available for finer 501(c) subclassification.

File Conventions

Every year/form directory contains two artifacts on S3:

processed/core/{YYYY}/{FORM_TYPE}/
    core_{YYYY}_{FORM_TYPE}.csv             # the data
    core_{YYYY}_{FORM_TYPE}_dictionary.csv  # variable definitions for this year

Quality reports are published on GitHub Pages alongside the CORE Pipeline Guide and linked from the Quality Report column in each table above. The Quality Reports Index lists all 109 published reports.

Data dictionaries are per-year because the variable set evolves as the underlying 990 forms change. Always read the dictionary for the year you’re using rather than assuming column stability across years.

Filter on is_501c3 (boolean) to subset to 501(c)(3) public charities or their complement. The raw subsection_cd (IRC EO subsection code, 1–93) is also present for finer breakdowns.

Tables above start in 2010 for the 990/990EZ/990 Combined forms and in 2010 for 990PF — earlier years exist as placeholder files only. For pre-2010 coverage, use the Legacy: Harmonized Core section below.



Before you analyze

Known upstream-IRS data shapes that look like pipeline bugs but aren’t — documented in the pipeline guide so analysts don’t have to rediscover them:

  • No top-coding or winsorization — identical maxes across related columns (e.g. total_contributions = total_revenue) reflect real-world equality for one filer, not capping.
  • ~12% of 990-EZ rows fail the balance-sheet identity — filer-supplied, not a harmonization error.
  • Schedule A _sec170 / _sec509 zeros are filer-test-specific — cross-reference non_pf_status_reason before interpreting them as missing data.
  • Filer-supplied negatives on revenue fields — IRS-supplied as-filed; not amendments.

Full pipeline guide and per-form quality reports are at the CORE Pipeline Guide.

Legacy: Harmonized Core (deprecated)

Show legacy harmonized/core/ files (for reproducibility only — use the by-form-type tables above)

These files predate the current by-form-type layout. They split each year’s filers along two axes — tax-exempt class (501c3 charities vs. all other 501c types) and form scope (PC = full 990 only, PZ = 990 + 990EZ) — producing five products. The current pipeline pools these into single per-form-type files with an is_501c3 column for filtering, so the products below are kept only to support replication of analyses built on the older layout.
