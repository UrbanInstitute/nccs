Core Data Catalog

The NCCS Core Data Series is a panel of nonprofit organizations derived from annual IRS Form 990 filings. This catalog organizes the published files by form type — pick the variant whose filing scope matches your analysis.

For background on what the Core Series contains and its scope limitations, see the Core overview page.

Which dataset should I use?

If you need… Use Why
Full Form 990 filers only, max variables 990 Larger nonprofits filing the long form (≈300 variables). Best for detailed financial analysis.
Short-form filers only 990EZ Smaller nonprofits filing the short form. Limited variable set.
Every 990 + 990EZ filer in one file 990 Combined The broadest population per year. Stacks 990 and 990EZ filers using the shared variable subset.
Private foundations 990PF Filed by all private foundations regardless of size. Separate variable set from public charities.

Each file pools 501(c)(3) charities and all other 501(c) types into a single product. Use the tscope column to filter to a specific tax-exempt class post-download.

File Conventions

Every year/form directory contains two artifacts on S3:

processed/core/{YYYY}/{FORM_TYPE}/
    core_{YYYY}_{FORM_TYPE}.csv             # the data
    core_{YYYY}_{FORM_TYPE}_dictionary.csv  # variable definitions for this year

Quality reports are not served from S3 (the S3 copies are gzipped). They are published on the nccs-data-core Pages site and linked from the Quality Report column in each table above.

Data dictionaries are per-year because the variable set evolves as the underlying 990 forms change. Always read the dictionary for the year you’re using rather than assuming column stability across years.

The tscope column distinguishes 501(c)(3) charities (CHARITIES) from all other 501(c) types (NONPROFIT) within each file. Filter on this column when your analysis needs charities only or excludes them.



Legacy: Harmonized Core (deprecated)

Show legacy harmonized/core/ files (for reproducibility only — use the by-form-type tables above)

These files predate the current by-form-type layout. They split each year’s filers along two axes — tax-exempt class (501c3 charities vs. all other 501c types) and form scope (PC = full 990 only, PZ = 990 + 990EZ) — producing five products. The current pipeline pools these into single per-form-type files with a tscope column for filtering, so the products below are kept only to support replication of analyses built on the older layout.
